          Dev Agent（代码开发 Agent）
          阶段 4 - 最核心的 Agent 实现
          核心阶段
          AGENTS.md 是 Agent 的"地图 + 防错手册"，每行对应一个历史失败案例
          L1 信息边界：角色设定 + 上下文裁剪 + 项目知识注入
          L2 工具系统：IDE / Git API / CI 触发 / RAG 知识库
          L3 执行编排：功能点拉取 → 依赖分析 → 编写代码 → 自检
          L4 状态管理：任务状态 JSON + Context Reset（利用率 > 60%）
          L5 评估：Lint + 类型检查 + 覆盖率 ≥ 80% + SAST 安全扫描
          L6 约束：单 PR 单功能 + 提交格式约束 + 安全红线
          覆盖率 ≥ 80% SAST = 0 高危 TL 审批
          🧪
          QA Agent（测试 Agent）
          阶段 8 + 9 - 用例产出 & 执行
          质量保障
          QA Planner Agent（阶段 8）：PRD → 完整测试用例库
          QA Exec Agent（阶段 9）：执行用例 + 自动提单 + 回归验证
          PRD 功能覆盖率检查：每个功能点必须有对应用例
          等价类完整性：正向 / 逆向 / 边界三类用例齐全
          P0 缺陷策略：发现 P0 立即暂停测试并通知 PM + TL
          P0 用例 ≥ 10 覆盖率 100% QA Lead 审批
          🔍
          Review Agent（代码审查 Agent）
          阶段 5 - 自动化 Code Review
          多维度评分：代码质量 / 安全性 / 可维护性 / 测试充分性
          问题分级：Blocker / Major / Minor / Suggestion
          通过标准：Blocker = 0，Major ≤ 2，否则无法合并
          可生成自动化修复建议，Agent 可直接采纳
          Blocker = 0 TL 确认
          🚀
          Deploy Agent（部署 Agent）
          阶段 10 - CI/CD 全链路自动化
          完整流水线：构建 → 镜像扫描 → Staging → 灰度 → 全量
          灰度策略：5% → 30% → 100%，期间错误率超标自动回滚
          自动回滚触发：错误率 > 1% 或 P95 > 500ms
          部署时间窗口：仅工作日 10:00-17:00
          Gate：PM + TL + QA 三方审批
          自动回滚 灰度发布 三方审批
